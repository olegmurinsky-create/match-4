## Intent

**Problem:** В конце уровня, когда целевой счет (targetScore) достигнут, игра теряет темп, так как игрок должен мучительно искать последние доступные ходы, чтобы завершить уровень.
**Approach:** Внедрить механику "Fever Mode": как только `score >= targetScore`, запускается 20-секундный таймер и активируется Fever Mode (с множителем очков). Когда таймер истекает, уровень автоматически завершается победой.

## Boundaries & Constraints

**Always:**
- Цель для следующего уровня (`targetScore`) должна рассчитываться на основе текущего счета, чтобы обеспечить прогрессивную сложность: `score + (nextLevel * 1000 - 500)`.
- Таймер должен стартовать ровно один раз за уровень при достижении targetScore.
- Множитель очков в Fever Mode должен быть равен 2x (базовые очки * 2).
- По истечении 20 секунд игра переходит в состояние `level_clear`.
- Обновления состояния React (в частности таймера) не должны вызывать лаги в `framer-motion` анимациях.
- Таймер должен очищаться при переходе на следующий уровень или рестарте.

**Ask First:**
- Если потребуется добавлять сложные анимации таймера, способные заблокировать UI поток.

**Never:**
- Не изменять базовую механику поиска совпадений или гравитацию.
- Не завершать уровень до истечения 20 секунд в Fever Mode (даже если ходов больше нет - пусть таймер дотикает, игрок должен чувствовать, что он в бонусном режиме).

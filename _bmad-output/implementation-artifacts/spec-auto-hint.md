## Intent

**Problem:** Игроки могут застрять на уровне, не видя доступных ходов, что приводит к разочарованию и прерыванию игровой сессии.
**Approach:** Внедрить пассивную систему подсказок. Если игрок бездействует в течение 5 секунд, два шарика, составляющие валидный ход, начинают слегка покачиваться, привлекая внимание. Анимация прекращается после хода игрока.

## Boundaries & Constraints

**Always:**
- Таймер бездействия должен сбрасываться только при клике на шарик (вызове `handleCellClick`). Простое движение мыши по экрану без клика не должно влиять на таймер.
- Анимация-подсказка должна быть ненавязчивой (легкое "потряхивание").
- Подсказка должна показывать только один из возможных ходов.

**Ask First:**
- Если реализация поиска хода окажется слишком ресурсоемкой и будет вызывать задержки в UI.

**Never:**
- Не добавлять в UI никаких кнопок для вызова подсказки.
- Не делать подсказку слишком очевидной или постоянной, чтобы не снижать сложность игры.

## I/O & Edge-Case Matrix

| Scenario | Input / State | Expected Output / Behavior | Error Handling |
|----------|--------------|---------------------------|----------------|
| Игрок бездействует | `gameState: 'playing'`, нет кликов в течение 5 секунд | Находятся 2 шарика для хода. Их `id` добавляются в `hintedBallIds`. Начинается анимация. | Если ходов нет, ничего не происходит. |
| Игрок делает ход | Пользователь кликает на любой шарик | Таймер бездействия сбрасывается. `hintedBallIds` очищается, анимация прекращается. | N/A |
| Подсказка во время каскада | `isProcessing: true` | Таймер бездействия не должен запускаться. | N/A |

**Acceptance Criteria:**
- Given игрок находится в состоянии 'playing' и не кликает на шарики, when проходит 5 секунд бездействия, then два шарика для возможного хода начинают анимироваться.
- Given анимация подсказки активна, when игрок кликает на любой шарик, then анимация немедленно прекращается, и таймер бездействия сбрасывается.
- Given игрок перемещает курсор мыши по экрану, but не кликает, when таймер бездействия запущен, then таймер не сбрасывается.
- Given идет обработка совпадений и каскадов (`isProcessing` is true), then таймер бездействия не должен запускаться.
